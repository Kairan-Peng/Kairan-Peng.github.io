【Java】网络编程（TCP，UDP）

发表于2022-04-03|更新于2022-04-05|java
|阅读量:

基础知识

端口

范围：0~65535

TCP、UDP总端口数：65536 * 2（单个协议下，端口号不能冲突）

端口分类

  • 公有端口：0~1023
    • HTTP：80
    • HTTPS：443
    • FTP：21
    • Telent：23
  • 程序注册端口：1024~49151（分配用户或者程序）
    • Tomcat：8080
    • MySQL：3306
    • Oracle：1521
  • 动态、私有：49152~ 65535

TCP

服务器：指定端口号，创建ServerSocket对象，用于监听端口

客户端：指定目的 IP 和端口号，创建Socket对象，创建对象的过程就是建立连接的过程。

Socket 对象好比是一个管道，一方是客户端，一方是服务器。

传输文字

服务器

  1. 创建一个ServerSocket / 创建一个端口监听
  2. 监听对该套接字的连接（阻塞）/ 开启这个端口监听
  3. 读取该连接socket的消息
  4. 关闭资源

客户端

  1. 指定服务器的ip地址和端口号
  2. 创建一个socket连接
  3. 发送消息IO流
  4. 关闭资源

传输文件

服务器

  1. 创建ServerSocket，监听端口
  2. 开始监听（阻塞）
  3. 获得监听到的socket的输入流
  4. 创建文件的输出流
  5. 将socket输入流写到文件输出流中
  6. 关闭资源

客户端

  1. 创建Socket
  2. 创建socket输出流
  3. 创建要传输文件的输入流
  4. 将文件输入流写到socket输出流中
  5. 关闭客户端到服务器的单向连接，不能输出但仍可接受服务器的数据
  6. 关闭资源

UDP

简单流程

用户A（发送方）

  1. 创建 DatagramSocket 对象
  2. 创建 DatagramPacket 对象（数据报包，包含数据和目的地址）
  3. 发送数据报包
  4. 关闭流

用户B（接收方）

  1. 创建 DatagramSocket 对象，绑定端口
  2. 创建 DatagramPacket 对象，接受数据会存到 buffer 里
  3. 开始监听（阻塞）
  4. 关闭流
