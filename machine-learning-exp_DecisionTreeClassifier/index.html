【机器学习】实践-使用决策树来预测Titanic幸存者

发表于2021-12-06|更新于2021-12-06|machine-learning
|阅读量:

1 数据准备

1.1 数据集下载

在kaggle上下载Titanic数据集。

1.2 读取数据并预处理

输出数据集的形状和列名：

(891, 11)
Index(['Survived', 'Pclass', 'Name', 'Sex', 'Age', 'SibSp', 'Parch', 'Ticket',
       'Fare', 'Cabin', 'Embarked'],
      dtype='object')

1.3 划分训练集和验证集

输出训练集和验证集的大小：

train dataset: (712, 7), test dataset: (179, 7)

2 模型训练

2.1 模型训练和评估

输出训练集和验证集的得分：

train_score: 0.9859550561797753
test_score: 0.8212290502793296

训练集的得分远高于测试集的得分，可以看出模型过拟合了。

3 模型优化

3.1 参数max_depth优化

max_depth: 生成决策树的最大深度。

3.1.1 构造不同max_depth的决策树模型，分别计算评分

输出不同max_depth下模型的训练结果：

best max_depth: 4, best score: 0.8435754189944135

随着max_depth的增加，模型对训练集的得分也越来越高，说明决策树层数越高，决策树的学习能力越强，也越容易发生过拟合。

所以限制max_depth能增加模型的泛化能力。

3.1.2 引入交叉验证来计算评分

多次运行之前的全部代码会发现：每次得到的最好的max_depth都不相同，这是因为每次随机划分的数据集不同，所以引入交叉验证来判断最优的max_depth。

输出不同max_depth下模型的交叉验证平均的训练结果：

max_depth: 2, cv_score: 0.7710986267166042
max_depth: 3, cv_score: 0.8080774032459426
max_depth: 4, cv_score: 0.8081148564294631

max_depth: 12, cv_score: 0.7935830212234707
max_depth: 13, cv_score: 0.7778901373283394
max_depth: 14, cv_score: 0.7868664169787765
best max_depth: 7, best score: 0.8227715355805243

在模型其他参数都使用默认值时，max_depth参数为7时模型能获得最好的预测效果

3.1.3 用 GridSearchCV 类进行参数优化

输出最优的max_depth参数和对应的最优评分：

best score:0.8137827715355807

使用GridSearchCV获得的结果和直接用k折交叉验证获得的结果不同。

3.2 参数min_impurity_decrease优化

min_impurity_decrease: 打算划分一个内部结点时，只有当该结点上的不纯度不小于该参数指定的值时，才会对该结点进行划分。

用 GridSearchCV 类进行参数min_impurity_decrease的优化（这里没有限制max_depth）

输出最优的min_impurity_decrease参数和对应的最优评分：

best min_impurity_decrease:{'min_impurity_decrease': 0.01020408163265306}
best score:0.7990886392009988

输出param_grid中能得到best score的参数组合，输出对应的best score：
