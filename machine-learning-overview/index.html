【机器学习】概述

发表于2021-11-26|更新于2021-12-06|machine-learning

1 机器学习简介

1.1 什么是机器学习

机器学习是近20多年兴起的一门多领域交叉学科，涉及概率论、统计学、逼近论、凸分析、算法复杂度理论等多门学科。

机器学习理论主要是设计和分析一些让计算机可以自动“学习”的算法。

机器学习算法是一类从数据中自动分析获得规律，并利用规律对未知数据进行预测的算法。

因为学习算法中涉及了大量的统计学理论，机器学习与统计推断学联系尤为密切，也被称为统计学习理论。算法设计方面，机器学习理论关注可以实现的，行之有效的学习算法。很多推论问题属于无程序可循难度，所以部分的机器学习研究是开发容易处理的近似算法。

1.2 机器学习发展历程

1.2.1 诞生

人工智能的诞生（1943年~1956年）

  • John von Neumann在1930年加入了普林斯顿大学，在数学物理系任教，和阿兰·图灵是同事。

  • Warren McCulloch和Walter Pitts在1943年发表了人工智能领域的开篇之作，提出了人工神经网络模型。

  • Marvin Minsky和Dean Edmonds在1951年建造第一台神经网络计算机。

  • 1956年：John McCarthy从普林斯顿大学毕业后去达特茅斯学院工作，说服了Marvin Minsky和Claude Shannon在达特茅斯学院组织一个暑期研讨会，召集了对机器智能、人工神经网络和自动理论感兴趣的研究者，参加由IBM赞助的研讨会。

1.2.2 各类算法的发展历史

1.2.3 未来发展方向

  • 集成学习（Ensemble Learning）：集成学习是一种机器学习范式，多个学习器被训练来解决同一个问题。与普通的机器学习方法试图从训练数据中学习一个假设不同，集成方法通过构建并结合多个机器学习器来完成学习任务。
  • 可扩展机器学习（对大数据集、高维数据的学习等）：分为向上扩展（Scale-up，采用更大规模的机器）和向外扩展（Scale-out，采用分布式计算方式）。
  • 强化学习（Reinforcement Learning）：智能体（Agent）执行了某个动作（Action）后，环境（Environment）将会转换到一个新的状态（State），对于该新的状态环境会给出奖励信号（Reward，正奖励或者负奖励）。随后，智能体根据新的状态和环境反馈的奖励，按照一定的策略执行新的动作。
  • 迁移学习（Transfer Learning）：专注于存储已有问题的解决模型，并将其利用在其他不同但相关问题上。把一个领域（即源领域）的知识，迁移到另外一个领域（即目标领域），使得目标领域能够取得更好的学习效果。
  • 深度学习（Deep Learning）：建立在人工多层神经网络基础上的机器学习。
  • 深度概率学习：结合深度神经网络和概率神经网络（如贝叶斯神经网络），将网络中确定的权重值变成了分布，引入了不确定性，增加了模型的鲁棒性和可解释性。模型会对预测结果额外给出置信度，置信度低低话说明模型对预测结果不是很确定。运用到自动驾驶上，当置信度低时自动驾驶将切换到手动驾驶，让驾驶员进行决策。

1.3 机器学习应用

  • 数据挖掘

2 统计学习与机器学习

统计学习可以看作是经典的机器学习，严格强调低维空间的问题的统计推导。

  • 对象：具有一定统计规律性的各种类型的数据。

  • 目的：对数据进行预测和分析。

  • 方法：从给定的、有限的、用于学习的训练数据（training data） 集合出发，假设数据是独立同分布产生的；并且假设要学习的模型属于某个函数的集合，称为假设空间（hypothesis space）应用某个评价准则（evaluation criterion），从假设空间中选取一个最优模型，使它对已知的训练数据及未知的测试数据（test data）在给定的评价准则下有最优的预测； 最优模型的选取由算法实现。

2.1 统计学习分类

2.1.1 基本分类

  • 监督学习（Supervised Learning）：监督学习的每一个输入都有一个已定的输出结果（标签）。监督学习目的是学习一个由输入到输出的映射，称为模型。模型的集合就是假设空间（hypothesis space）。

  • 无监督学习（Unsupervised Learning）：对每个输入都没有标签。

  • 强化学习（Reinforcement Learning）
  • 半监督学习（Semi-supervised Learning）
  • 主动学习（Active Learning）

2.1.2 按模型分类

  • 概率模型与非概率模型
  • 线性模型与非线性模型
  • 参数化模型与非参数化模型

2.1.3 按算法分类

  • 在线学习（Online Learning）
  • 批量学习（Batch Learning）

2.1.4 按技巧分类

  • 贝叶斯学习（Bayesian Learning）
  • 核方法（Kernel Method）

2.2 统计学习三要素

2.2.1 模型

线性模型、非线形模型等，参数空间（假设空间，hypothesis space）。

类比深度学习里的模型model。

2.2.2 策略

统计学习的策略是使结构风险最小化。

一些概念：

  • 损失函数（loss function）：一次预测的好坏，$L(Y,f(X))$ ，预测值与实际值的差异。

  • 风险函数（risk function）：平均意义下模型预测的好坏

  • 经验损失/经验风险（empirical loss / empirical risk）：模型在训练集的表现情况

  • 经验风险最小化：在训练集中模型的风险最小（平均损失最小），此时模型对训练集中的数据有着最大的拟合程度。

  • 结构风险最小化（structure risk minimization）：为防止过拟合提出的策略，等价于正则化（regularization），即在经验风险中加入一个正则项（regularizer，或罚项 penalty term）。

类比深度学习里的损失函数loss。

2.2.3 算法

如果最优化问题有显式的解析式，算法比较简单。

但通常解析式不存在，就需要数值计算的方法。

类比深度学习里的优化器optimizer，随机梯度下降算法等。

3 泛化能力与模型和数据的关系

泛化能力是指模型对于未知数据也有着准确预测的能力。

了解泛化能力与模型之间的关系能帮助我们去选择模型。

3.1 泛化能力与模型的关系

模型的假设空间越大，即模型参数越多，模型越复杂，则

模型的学习能力越强，则

模型能更好的拟合训练集中的数据，

模型的泛化能力越差。

模型会将噪声学进去，会将不重要的特征学进去，导致对于未知的数据等预测能力下降，也就是过拟合（over-fitting）。

模型太简单会导致无法学习到数据的特征。

M=0, 1时，模型无法学习到数据的特征。

M=3时，有较好的拟合。

M=9时，模型过拟合。

（M为假设空间大小）

3.2 泛化能力与数据的关系

数据的特征越多，数据量越大，则

训练出来的模型泛化能力越强。

但会导致计算量增大，训练时间增大，且特征越多则需要的数据量越大（纬度的诅咒）。

4 正则化与交叉验证

4.1 正则化

正则化：在经验风险中加入一个正则项（regularizer，或罚项 penalty term） 𝞴𝑱(𝒇) ，构成结构风险，该项与模型复杂度有关，抑制模型的复杂程度，提高模型的泛化能力，抑制过度拟合。

4.2 交叉验证

交叉验证：将训练集的数据再次分为训练集和验证集，训练集参与模型的训练，验证集不参与训练，用作模型的评估与选择。

  • 简单交叉验证
  • S折交叉验证：将训练集均分成S份，其中S-1份参加训练，最后1份验证，可以有S种划分组合，对同一个模型进行这S种组合的评价。
  • 留一交叉验证：1个样本点作为验证集，其余为测试集，可以有样本数目大小的组合。

5 生成模型与判别模型

监督学习方法可以分为生成方法(generative approach)和判别方法(discriminative approach)，所学到的模型分别称为生成模型(generative model)和判别模型(discriminative model)。

  • 生成方法：可还原出联合概率分布$P(X,Y)$，而判别方法不能。生成方法的收敛速度更快，当样本容量增加的时候，学到的模型可以更快地收敛于真实模型；当存在隐变量时，仍可以使用生成方法，而判别方法则不能用。

  • 判别方法：直接学习到条件概率或决策函数，直接进行预测，往往学习的准确率更高；由于直接学习$Y=f(X)$或$P(Y|X)$，可对数据进行各种程度上的抽象、定义特征并使用特征，因此可以简化学习过程。

监督学习的目的就是学习一个模型：

决策函数：$Y=f(X)$

条件概率分布：$P(Y|X)$

5.1 生成模型

生成方法由数据学习输入和输出联合概率分布$P(X,Y)$，然后求出后验概率分布𝑷 (𝒀|𝑿)作为预测的模型，即生成模型。

生成方法（Generative approach）对应生成模型：$P(Y|X)=\frac{P(X,Y)}{P(X)}$

如：

朴素贝叶斯法、隐马尔科夫模型、高斯混合模型。

5.2 判别模型

判别方法由数据直接学习决策函数$Y=f(X)$或者条件概率分布$P(Y|X)$作为预测的模型，即判别模型。判别方法关心的是对给定输入，应该预测什么样的输出y 。

判别方法（Discriminative approach）对应discriminative model：

$Y=f(X)$ 或$P(Y|X)$

如：

K近邻法（KNN）、感知机（Perceptron）、决策树、logistic回归模型、最大熵模型、支持向量机（SVM）、提升方法和条件随机场（CRF）、神经网络、boosting。

6 问题的分类

将实际需要解决的问题抽象后，可以有分为以下几类：

  • 分类问题
  • 标注问题
  • 回归问题

6.1 分类问题

比如判断图片里的主体是否是猫（二分类问题），判断图片的主体是什么动物（多分类问题）等。

6.2 标注问题

标注问题简单来说就是：输入观测序列，输出标记序列或状态序列。

训练集：$T=\{(x_1,y_1),(x_2,y_2),…,(x_N,y_N)\}$

观测序列：$x_i=(x_i^{(1)},x_i^{(2)},…,x_i^{(n)})^T,i=1,2,…,N$

输出标记序列：$y_i=(y_i^{(1)},y_i^{(2)},…,y_i^{(n)})^T$

模型：条件概率分布$P(Y^{(1)},Y^{(2)},…,Y^{(n)}|X^{(1)},X^{(2)},…,X^{(n)})$

例子：

标记表示名词短语的“开始”、“结束”或“其他”（分别以B, E, O表示) 。

输入：At Microsoft Research, we have an insatiable curiosity and the desire to create new technology that will help define the computing experience.

输出：At/O Microsoft/B Research/E, we/O have/O an/O insatiable/6 curiosity/E and/O the/O desire/BE to/O create/O new/B technology/E that/O will/O help/O define/O the/O computing/B experience/E.

6.3 回归问题

回归模型是表示从输入变量到输出变量之间映射的函数。回归问题的学习等价于函数拟合。

常用损失函数是MSE（平均平方误差），或称最小二乘法。

常见回归为线性回归。

6.3.1 一维线性回归

如上图所示一维线性最小二乘，获取一条拟合直线，使得对每个点在直线上的估计值与实际值的距离的平方和最小。拟合直线为$\hat f(x)=w_0+w_1x^{(1)}$

6.3.2 二维线性回归

二维线性回归如下图所示，拟合平面为$\hat f(x)=w_0+w_1x^{(1)}+w_2x^{(2)}$

对上图问题而言，二次函数拟合度更好，如下图所示，拟合曲面为$\hat f(x)=w_0+w_1x^{(1)}+w_2x^{(2)}+w_3x^{(1)^2}+w_4x^{(2)^2}$

6.3.3 基函数线性回归

针对上述无法线性拟合，需要二次函数拟合的问题，提出基函数线性回归。

基函数线性回归将输入𝑿的每一个维度，用其他函数形式代替，这个用于替代的函数则被称为基函数。

引入基函数后，非线性回归问题转换为了线性回归问题。

基函数线性回归模型：

$f_w(x) = \sum_{j=1}^M w_j \phi_j(x) \ \ \ \ \ \ \ \ w=(w_1,…,w_M)^T, \phi=(\phi_1,…,\phi_M)^T$

一维向量基函数：$\phi(x)=(\phi_1(x),…,\phi_M(x))^T$

  • 多项式形式：$\phi(x) = (1,x,x^2,…,x^{M-1})^T$

  • 三角多项式：$\phi(x) = (1,sinx,cosx,sin2x,cos2x,…,sinmx,cosmx)^T$

  • 高斯核基函数：$\phi_j(x) = exp\{-\frac{(x-\mu_j)^2}{2s^2}\}$

  • Sigmoidal基函数：$\phi_j(x) = \sigma(\frac{x-\mu_j}{s})$

7 分类问题的评价指标

7.1 精准率和召回率

二分类的评价指标。

  • TP（true positive）：预测为正样本，预测正确，即实际为正样本。
  • FN（false negative）：预测为负样本，预测错误，即实际为正样本。
  • FP（false positive）：预测为正样本，预测错误，即实际为负样本。
  • TN（true negative）：预测为负样本，预测正确，即实际为负样本。
  • 查准率（Precision）： $P = \frac{TP}{TP+FP}$ ，分子为实际为正样本、预测为正样本的数量，分母为预测为正样本的数量。反映预测为正样本的数据的实际正确率。
  • 召回率（Recall，查全率）： $R = \frac{TP}{TP+FN}$ ，分子为实际为正样本、预测为正样本的数量，分母为实际为正样本的数量。反映所有实际为正样本的数据的预测正确率。
  • $F_1$ 值： $\frac{2}{F_1} = \frac{1}{P} + \frac{1}{R}$ ，同时兼顾了分类模型的查确率和召回率，可以看作二者的调和平均。

7.2 混淆矩阵

混淆矩阵（Confusion Matrix）:多分类的评价指标。

混淆矩阵的每一列代表了预测类别，每一列的总数表示预测为该类别的数据的数目。每一行代表了数据的真实归属类别，每一行的数据总数表示该类别的数据实例的数目。每一列中的数值表示真实数据被预测为该类的数目。

类1 43 2 0
类2 5 45 1
类3 2 3 49

7.3 ROC

ROC的全名叫做Receiver Operating Characteristic，中文名字叫“受试者工作特征曲线”，其主要分析工具是一个画在二维平面上的曲线——ROC 曲线。平面的横坐标是false positive rate(FPR)，纵坐标是true positive rate(TPR)。对某个分类器而言，我们可以根据其在测试样本上的表现得到一个TPR和FPR点对。这样，此分类器就可以映射成ROC平面上的一个点。调整这个分类器分类时候使用的阈值，我们就可以得到一个经过(0, 0)，(1, 1)的曲线，这就是此分类器的ROC曲线。

  • FPR（false positive rate，假阳性率）：$FPR = \frac{FP}{FP+TN}$ ，分母是实际负样本的个数，分子是负样本中被分类器预测为正样本的个数。
  • TPR（true positive rate，真阳性率）： $TPR = \frac{TP}{TP+FN}$ ，分母是实际正样本的个数，分子是正样本中被分类器预测为正样本的个数。
  • AUC（Area Under Curve）：为ROC曲线下的面积，取值范围一般在0.5和1之间。使用AUC值作为评价标准是因为很多时候ROC曲线并不能清晰的说明哪个分类器的效果更好，而作为一个数值，对应AUC更大的分类器效果更好。

8 梯度下降

梯度下降算法用于模型训练中对模型的参数（权重）$w$进行更新。

梯度是指损失函数$Loss$对$w$的偏导数，往梯度下降的方向更新$w$，找到损失函数最小的点，达到拟合的目的。

8.1 随机梯度下降算法

如果用训练集所有数据来计算平均损失值，会遇到以下问题：

  • 计算量大，每次更新都需要计算训练集所有的损失
  • 容易遇到鞍点，导致梯度消失，无法继续更新权重
  • 容易遇到局部最优点，而不是全局最优点

随机梯度下降算法就是用于应对这些问题。

随机梯度下降算法每次更新权重时，不是以训练集所有数据，而是随机选取训练集中的一个或多个数据来更新。

少量数据里的噪声有助于跳出鞍点和局部最优点。

大量实践证明这是十分有效的方法。

9 模型选择流程

  1. 准备模型的候选$M_1,M_2,…,M_k$

  2. 对各个模型$M_1,M_2,…,M_k$求解其学习结果$f^{(1)},f^{(2)},…,f^{(k)}$

  3. 对各个学习结果$f^{(1)},f^{(2)},…,f^{(k)}$的泛化误差$G^{(1)},G^{(2)},…,G^{(k)}$进行评价

  4. 选择泛化误差$G^{(1)},G^{(2)},…,G^{(k)}$最小的模型作为最终模型

泛化误差（Generalization Error）：模型在真实情况下的误差，在交叉验证中表现为模型在验证集中的损失，在模型评审中表现为模型在测试集中的损失。
